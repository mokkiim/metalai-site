# Vibe Coding (바이브 코딩)

## 정의
개발자가 **설계·구현의 거의 전부를 AI 에이전트에 위임**하고, 자신은 준기술 관리자 역할만 수행하는 코딩 방식.

## 특징
- 프롬프트로 기능을 기술하고 AI가 전체 파일·아키텍처를 생성
- 인간은 **테스트 통과 여부**와 **느낌**만으로 수용 여부 판단

## 바이브 코딩이 실패하는 이유
1. **기본 원칙 무시**: 아키텍처·경계·책임 분리 없이 버그 기하급수 증가
2. **핵심 설계 결정 미루기**: 누적되어 명확한 사고를 방해
3. **테스트의 잘못된 안도감**: 500+ 테스트도 예외 못 잡음
4. **맥락 상실**: AI가 만든 코드는 "첫날부터 레거시"

> **작동하는 코드 ≠ 유지보수 가능한 코드**
